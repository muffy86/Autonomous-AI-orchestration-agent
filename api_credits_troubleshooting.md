# Browser Use API Credits Troubleshooting Guide

## Why Your API Credits Show Zero

Your Browser Use account is showing zero API credits/tokens. Here's a comprehensive guide to understand and resolve this issue.

---

## Possible Reasons

### 1. **No API Credits Included in Your Plan**
Browser Use Premium may focus on browser automation features rather than API access.

**Action:**
- Visit https://browser-use.com/pricing
- Check what's included in "Browser Use Premium"
- Compare with plans that include API access
- You may have been paying for browser automation, not API credits

### 2. **Credits Already Consumed**
Even minimal usage can consume credits through:
- Background automation tasks
- Token processing (input + output tokens)
- Web search features (often charged separately)
- Connected third-party applications

**Action:**
- Log into cloud.browser-use.com
- Navigate to Billing → Usage or Activity Logs
- Review detailed breakdown of credit consumption
- Check for any automated tasks running in background

### 3. **Credit Expiration**
Many platforms expire unused credits after 12 months.

**Action:**
- Check your billing dashboard for "Credit Grants" section
- Look for expiration dates on any promotional credits
- Verify if your credits have "Expired" status
- Note that purchased credits may have different expiration rules than free trial credits

### 4. **Organization/Project Level Management**
Credits might be managed differently than you expect.

**Action:**
- Check if you're part of an organization account
- Verify you're viewing the correct project/API key dashboard
- Credits may be pooled at organization level, not individual account

### 5. **Negative Balance from Previous Usage**
If previous tasks exceeded your budget, it could create a negative balance.

**Action:**
- Review billing history for any overages
- Check if previous months showed negative charges
- New credits may have been used to cover old debts

### 6. **Minimum Balance Requirements**
Some platforms require minimum credit balance to function.

**Action:**
- Check if there's a minimum threshold requirement
- Verify auto-recharge settings
- See if manual recharge is needed
