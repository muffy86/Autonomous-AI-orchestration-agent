## 🔐 Required Secrets Configuration

To run the application, you need to configure the following API keys and tokens:

### 1. XAI API Key
- **What**: API key for xAI's Grok model (default chat model)
- **Where to get**: https://console.x.ai/
- **How to add**: 
  - For Cloud Agents: Add to [Cursor Dashboard > Cloud Agents > Secrets](https://cursor.com/settings)
  - For local development: Add to `.env` file

### 2. Vercel Blob Storage Token
- **What**: Token for file storage functionality
- **Where to get**: https://vercel.com/docs/storage/vercel-blob
- **How to add**: Same as above

### 3. PostgreSQL Database URL
- **What**: Connection string for the Postgres database
- **Where to get**: https://vercel.com/docs/storage/vercel-postgres/quickstart
- **Format**: `postgresql://user:password@host:port/database`
- **Local alternative**: Set up local Postgres with Docker:
  ```bash
  docker-compose up -d
  # This will start a local Postgres instance
  # Default URL: postgresql://postgres:postgres@localhost:5432/chatbot
  ```

### 4. Redis URL (Optional)
- **What**: Connection string for Redis (rate limiting, caching)
- **Where to get**: https://vercel.com/docs/redis
- **Format**: `redis://default:password@host:port`
- **Note**: Application can run without Redis for basic functionality

## 🆘 Troubleshooting

### "Cannot find module" errors
```bash
rm -rf node_modules pnpm-lock.yaml
pnpm install
```

### Database connection errors
- Verify `POSTGRES_URL` is correct
- Check database is running: `docker-compose ps`
- Run migrations: `pnpm db:migrate`

### Port 3000 already in use
```bash
# Find and kill the process
lsof -ti:3000 | xargs kill -9
# Or use a different port
PORT=3001 pnpm dev
```

### TypeScript errors
```bash
# Clear Next.js cache
rm -rf .next
pnpm build
```
